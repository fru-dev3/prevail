Why council beats a single model

Single-model output is fine for routine asks. For decisions where the cost of being wrong is real, one AI gives you one viewpoint shaped by one company's data and one RLHF bias. You don't know what you're missing.

Triangulation, not averaging

When Claude, Codex, and Gemini converge — that's real signal. When they disagree, the disagreement is the trade-off you needed to think about. A single model would have papered over it.

Cancels per-model quirks

Claude hedges, Codex is conservative on non-code, Gemini is verbose. The synthesizer chair reads all three and writes one direct answer with none of any single model's stylistic baggage.

Compare versions side-by-side

Run Opus 4.7 and 4.6 in the same panel. Watch them disagree. Find out which one actually performs better on your kind of question, on questions about your actual life.

Your data, your machine

Every chat runs against a folder of markdown — your vault, on your disk. No cloud sync, no app server, no telemetry. Bring your own CLI auth and your data never leaves the machine.

Install

macOS (Apple Silicon, Intel) and Linux (x64, arm64) — single self-contained binary, no Node or Bun required at runtime.

curl -fsSL prevail.ai/install | bash

or from source: git clone https://github.com/fru-dev3/prevail && cd prevail && bun install && bun run start

First boot opens a wizard: pick a vault (bring your own, point at ~/.ai/vault, or accept the bundled synthetic demo). Then arrow up/down through your domains and start a chat.
